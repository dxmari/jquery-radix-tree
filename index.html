Guides: Usage & Demos
Basic Example
With Radix Tree, you can render simple tree deeply nestable nodes with a basic hierarchical data structure. You can expand and collapse nodes, and the tree supports unlimited nesting. Useful for representing folders, categories, or any parent-child relationships.
For example, you can use import this snippet inside your files like this:
Basic Demo
              
  const data = [
    {
      label: 'Universe',
      open: true,
      children: [
        { label: 'Galaxies', children: [ { label: 'Milky Way' }, { label: 'Andromeda' } ] },
        { label: 'Black Holes' }
      ]
    }
  ];
  $('.radix-tree').radixTree({ data });

Lazy Loading
Load children on demand with async support. This example shows how to use the lazyLoad callback to fetch children only when a node is expanded. Ideal for large datasets or when loading data from a server.
For example, you can use import this snippet inside your files like this:
Lazy Loading Demo
              
  function myLazyLoad(node, done, delay) {
      setTimeout(() => {
        if (node.label === 'Galaxies') {
          done([{ label: 'Milky Way' }, { label: 'Andromeda' }]);
        } else {
          done([{ label: 'No data' }]);
        }
      }, delay); // Use the dynamic delay
    }
  $('.radix-tree').radixTree({
    data: [{ label: 'Galaxies', lazy: true }],
    lazyLoad: myLazyLoad,
    lazyLoadDelay: 1200 // Set delay dynamically (ms)
  });

Infinite Scroll
Efficiently browse huge folders or datasets. This example shows how to enable infinite scroll and pagination for large trees, loading more nodes as you scroll. Useful for file browsers or any large hierarchical data.
For example, you can use import this snippet inside your files like this:
Infinite Scroll Demo
              
  const demoData = [
      {
        label: 'Big Folder',
        open: true,
        infinite: true,
        lazy: true,
        badge: 100,
        tags: ['infinite', 'files']
      }
    ];
  function infiniteLazyLoad(node, done, opts, delay) {
    const total = 100;
    const page = opts && opts.page ? opts.page : 1;
    const pageSize = opts && opts.pageSize ? opts.pageSize : 20;
    const start = (page - 1) * pageSize;
    const end = Math.min(start + pageSize, total);
    const children = [];
    for (let i = start; i < end; i++) {
      children.push({
        label: 'File ' + (i + 1),
        badge: (i % 2 === 0) ? 'even' : 'odd',
        tags: (i % 10 === 0) ? ['milestone'] : []
      });
    }
    setTimeout(() => {
      done(children, end < total);
    }, delay); // Use the dynamic delay
  }
  // Example 1: Small pageSize, default threshold (pagination enabled for 5)
  $('.radix-tree-small').radixTree({
    data: demoData,
    lazyLoad: infiniteLazyLoad,
    pageSize: 5,         // Pagination enabled for 5 (default threshold is 5)
    lazyLoadDelay: 800
  });
  // Example 2: Custom threshold (pagination only for pageSize >= 10)
  $('.radix-tree-large').radixTree({
    data: demoData,
    lazyLoad: infiniteLazyLoad,
    pageSize: 5,         // Pagination NOT enabled for 5 (threshold is 10)
    paginateThreshold: 10,
    lazyLoadDelay: 800
  });


Output

Command API
Interact with the tree after initialization. This example covers the available API commands for getting and setting checked nodes, expanding/collapsing nodes, and updating the tree data programmatically.
For example, you can use import this snippet inside your files like this:
Command API Demo
              
  const $tree = $('.radix-tree');
  // Get checked nodes
  const checked = $tree.radixTree('getChecked');
  // Set checked state
  $tree.radixTree('setChecked', 'node-id', true);
  // Expand/collapse nodes
  $tree.radixTree('expand', 'node-id');
  $tree.radixTree('collapse', 'node-id');
  // Get/set data
  const data = $tree.radixTree('getData');
  $tree.radixTree('setData', newData);

Troubleshooting

Common issues and solutions when using the Radix Tree jQuery Plugin.

Tree not rendering?

Make sure you included jQuery and the plugin script before your initialization code. Check the browser console for JavaScript errors.

Checkboxes not working?

Check for JavaScript errors in the console and ensure your data structure is correct. Verify that nodes have the proper checked and indeterminate properties.

Custom classes not appearing?

Verify you are using the className and rootClassName options as shown in the examples. Check that your CSS is properly targeting the generated class names.

Lazy loading not working?

Ensure you've set lazy: true on nodes and provided a lazyLoad callback function. Check that the callback calls the done function with the children array.

Infinite scroll issues?

Verify that infinite: true is set and check your pageSize and lazyLoadDelay settings. Ensure your lazy load callback handles pagination correctly.

Contributing

We welcome pull requests and issues! Here's how you can contribute to the Radix Tree jQuery Plugin.

How to Contribute

  1. Fork the repo - Create your own copy of the repository
  2. Create a feature branch - Work on your changes in a separate branch
  3. Add tests if possible - Ensure your changes work correctly
  4. Open a PR - Submit a pull request with a clear description of your changes

Development Setup

  • Clone the repository: git clone https://github.com/yourusername/JM-Dashboard.git
  • Install dependencies: npm install (if you want to run tests)
  • Make your changes to tree.js and style.css
  • Test your changes by opening index.html in a browser

Testing

Optional Mocha/Chai tests are available for plugin logic. Run tests with:

npm test

Questions & Support

For questions, open an issue or start a discussion on GitHub. We're here to help!

Note: The plugin is designed to be lightweight and dependency-free (except jQuery). When contributing, please maintain this philosophy and ensure any new features don't add unnecessary complexity.
